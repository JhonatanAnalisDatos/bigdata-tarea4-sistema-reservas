# Sistema de Reservas para Restaurante - MongoDB

## Descripción del Proyecto
Implementación de un sistema completo de reservas para restaurante utilizando MongoDB como base de datos NoSQL para el curso Big Data de la UNAD.

## Objetivo
Demostrar las capacidades de MongoDB en el manejo de datos semi-estructurados y la ejecución de consultas complejas en un escenario real.

## Estructura de la Base de Datos

### Colecciones
1. **mesas** - Información de mesas disponibles
2. **clientes** - Datos de clientes registrados  
3. **reservas** - Registro de reservas realizadas

## Consultas Implementadas

### 1. Consultas Básicas (CRUD)
- Listado de mesas y clientes
- Consulta de reservas activas
- Conteo de documentos

### 2. Consultas con Filtros
- Reservas por número de personas
- Mesas por capacidad
- Reservas con requerimientos especiales

### 3. Consultas de Agregación
- Estadísticas de ocupación
- Horarios más populares
- Clientes más frecuentes
